# Leaflet 地图瓦片 1px 缝隙问题排查与技术探索文档

在本项目主地图服务中，用户在部分浏览器环境或高分屏下，使用硬件加速（`translate3d`）进行地图缩放及拖拽时，会观察到相邻瓦片之间出现 1 像素宽度的白色或灰色网格分割线（即 1px 缝隙问题）。

本文档记录了针对该问题的深度技术排查、多种方案的探索轨迹、对比分析以及最终的最优解决方案，以便后续维护与有迹可循。

---

## 1. 缝隙产生的物理成因

Leaflet 作为轻量级地图引擎，为了在平移和层级缩放（Zoom Transition）时获得极佳的流畅度，默认会使用 CSS 3D 硬件加速定位属性：
`transform: translate3d(x, y, 0)`

当浏览器渲染绝对定位的瓦片图片时：
- 如果由于层级缩放、高清屏（Retina）亚像素计算、或浏览器本身非 100% 比例缩放等原因，计算出来的坐标 `x` 和 `y` 包含浮点数（例如 `300.5px`）；
- 浏览器在光栅化与像素对齐时会采用**亚像素插值渲染**（Subpixel Rendering）；
- 在图片最边缘处，插值器会由于没有相邻的图片源数据，转而将图片边缘像素与**地图容器的底层背景色**进行混合（Leaflet 容器默认背景通常为浅灰 `#ddd` 或白色）。
这导致相邻图片交界处发生极微弱的背景色显露，视觉上即呈现出一条淡淡的网格白缝。

---

## 2. 探索轨迹与方案对比分析

在解决此问题的过程中，我们进行了一系列由浅入深的方案尝试：

### 方案 1：CSS 图像抗锯齿与 outline 修复
- **配置方式**：
  ```css
  .leaflet-tile {
    outline: 1px solid transparent;
    -webkit-backface-visibility: hidden;
    backface-visibility: hidden;
  }
  ```
- **测试结果**：无法彻底消除分割白线，且在部分深色底图（如卫星图）交界处反而因为渲染器的亚像素边缘处理，生成了一条细微的灰色描边，使网格在视觉上更加突兀。

### 方案 2：百分比放大拉伸 `width: calc(100% + 1px) !important`
- **配置方式**：在 CSS 中强制将瓦片图片的长宽比定位大小扩充 1 像素。
- **测试结果**：**导致地图彻底黑屏/白屏无法渲染**。
- **成因分析**：Leaflet 所有图片瓦片并不是独立包裹在小方块容器中，而是扁平、并排地挂在同一个大图层容器 `.leaflet-tile-container` 之下。大图层容器在渲染时并没有具体的 `width` 限制（为 `auto` 或 `0`）。因此，如果强制对图片应用 `100%` 大小，图片在向下寻找父容器宽度计算时会发生宽度坍塌，实际宽度变为了 `0`，从而使得所有瓦片图像彻底消失。

### 方案 3：全局禁用 3D 硬件加速 `L.Browser.any3d = false`
- **配置方式**：在地图入口 JS 逻辑中将 Leaflet 全局 3D 支持置为 `false`，强行令其回退至 2D 的 `left`/`top` 绝对整数像素对齐。
- **测试结果**：缝隙确实被完美消灭。但是**直接导致 Leaflet 的缩放过渡平滑动画（Zoom Animation）失效**。每次缩放层级时，地图都会硬生生先白屏清空，然后再重新生成瓦片出现，严重降低了缩放交互体验。因违背“必须保留平滑缩放过渡”的硬性体验要求，被驳回。

### 方案 4：现代 CSS 独立 `scale` 属性或动态 JS 改变宽高像素
- **配置方式**：
  - CSS 中给图片加 `scale: 1.005 !important`（不覆盖行内 transform 定位）；
  - 或在 JS 中拦截 `L.TileLayer.prototype.createTile` 赋值，在计算出的具体行内像素上（如 256px 变为 257px）加 1 像素。
- **测试结果**：3D 缩放平滑过渡被保留，但**分割线在视觉上反而显得更加宽大和明显**。
- **成因分析**：在 3D 加速定位下，一旦图片本身被进行了哪怕微弱的拉伸或缩放（不管是 scale 比例放大还是像素级宽高改变），都必然会触发浏览器的**图像纹理过滤与重采样插值（Texture Filtering）**。在插值过程中，由于图片边缘没有相邻元素延伸，最外圈像素的 Alpha 透明度会发生淡化衰减，从而更严重地透出了底层背景，反向加深了白线印记。

---

## 3. 终极最优解：负外边距无损重叠方案 (Margin Overlap)

为了在保留 3D 位移与缩放动画的前提下，不破坏任何图片原始宽高比例（避免插值滤镜引起的泛白与模糊），我们最终采用了**负外边距无损重叠方案**。

### CSS 规则配置
```css
.leaflet-tile-container img {
  /* 使用 1px 负外边距，在不拉伸图片（无 scale、保持 256px/512px 原生宽高点对点画质）的前提下，
     让绝对定位的瓦片在渲染边界上自然产生 1px 的物理重叠，完美遮盖亚像素缝隙，且 3D 动画毫无影响 */
  margin: -1px !important;
}
```

### 运行机制与优势
1. **点对点原生清晰度**：瓦片图片的大小（`width`/`height`）依然为原生的 256px 或 Retina 下的 512px。由于图片**没有任何缩放或拉伸**，边缘的纹理采样极其纯净，绝对不会产生因为抗锯齿插值导致边缘泛白或模糊的残影。
2. **自然边界外溢**：在 CSS 盒模型中，绝对定位的图片在设置了 `margin: -1px !important` 后，它的物理渲染原点会向左上角微移 1 像素，同时其整体渲染边缘会在不改变定位坐标的前提下，向四周外扩 1px 的多余渲染空间。
3. **天衣无缝重叠**：相邻的瓦片因此在数学定位上产生了刚好 1 像素的重叠区域。这 1 像素的物理重叠完全将浮点数舍入计算产生的亚像素空隙挡住，白线彻底消失。
4. **性能与体验无损**：由于没有禁用 any3d，`translate3d` 与缩放层级时的过渡淡入淡出动画完整保留，地图操作依然保持最高性能的柔和与流畅。
