# 地图桌面旋转与预缓存任务操作优化需求

## 背景

当前地图旋转主要依赖移动端手势，桌面端缺少直接的按键辅助旋转体验。管理后台预缓存任务也存在三个操作限制：未完成任务不显示预览入口，删除任务只能移除任务记录，且预缓存并发上限偏低。

## 目标

- 桌面端地图支持按住 `Shift` 后通过鼠标操作旋转地图，普通拖拽仍保持平移。
- 预缓存任务在未完成时也可以预览任务区域，便于检查已缓存范围和当前任务位置。
- 删除预缓存任务时，管理员可以选择是否同时删除该任务区域和图层对应的瓦片缓存文件。
- 预缓存并发上限从当前偏低值调高，满足更大规模本地预缓存任务的执行需求。

## 行为约定

- 桌面端保留 `leaflet-rotate` 的 `Shift + 滚轮` 旋转能力，并新增 `Shift + 鼠标拖拽` 旋转能力。
- 预缓存任务只要仍在任务列表中且未处于删除中，就应展示预览入口。
- 删除任务默认只删除任务记录；勾选“同时删除关联缓存文件”后，后端应按任务的 provider、bounds、zoom 范围计算瓦片 URL，并删除对应 fetch-relay 缓存文件和元数据。
- 正在执行的任务被删除时应先停止任务；若同时选择删除缓存文件，应在任务停止后清理关联缓存，避免正在写入的瓦片残留。
- 默认预缓存最大并发上限调高到 64，前端输入限制与后端配置保持一致。

## 非目标

- 不改变瓦片缓存目录结构。
- 不引入独立任务队列服务。
- 不改变预缓存任务的瓦片数量建议上限语义。
