# 地图分享、KML 标注工作流与访问密码调整需求

## 背景

当前前台地图已经支持视图坐标、缩放和旋转角度写入 URL，也支持 KML 导入、点位新增和访问密码控制。但在实际使用中仍有几处体验断点：

- 分享链接无法保留当前底图图层，接收方打开后看到的图层可能与分享者不同。
- KML 点位新增入口依赖已导入文件列表中的新增按钮，移动端现场标注不够直接。
- 没有默认 KML 文件，未导入文件时无法直接保存长按新增的点位。
- KML 文件只能通过导入产生，缺少空白文件的新建能力。
- KML 文件名称不能重命名。
- 管理后台设置前台访问密码要求至少 10 位，和轻量私有地图的使用场景不匹配。
- 右下角工具按钮过多，核心按钮区域需要收纳低频入口。

## 目标

1. URL 分享应包含当前图层参数，打开链接后能恢复相同底图。
2. KML 应始终存在一个默认文件，用于保存快速新增的标注点。
3. 用户长按地图位置时可创建 KML 点位。
4. 多个 KML 文件存在时，新增点位弹窗应允许选择保存目标，并记住上次选择。
5. 用户可在 KML 面板中新建空白 KML 文件。
6. 用户可重命名 KML 文件。
7. 前台访问密码最小长度调整为 4 位。
8. 右下角入口将图层、KML 管理、管理后台收纳到同一个“更多/图层”按钮下。

## 交互要求

### 图层 URL 状态

- URL 使用 `layer` 参数记录当前底图名称。
- 写入坐标状态时保留现有 URL 参数，避免覆盖 `layer` 等状态。
- 当 URL 中存在合法 `layer` 时，优先按 URL 恢复；否则使用本地上次图层；再否则使用默认图层。

### KML 默认文件和新建文件

- 初始化 KML 模块时自动保证存在一个默认 KML 文件。
- 默认 KML 文件不能被删除，避免长按新增点位没有落点。
- KML 面板提供新建空白 KML 文件入口。
- 导入 KML 仍保留原流程。

### 长按地图新增点位

- 用户在地图空白位置长按后打开“新增地图标注”弹窗。
- 仅默认文件存在时，点位直接保存到默认文件。
- 多个可用 KML 文件存在时，弹窗提供保存目标选择。
- 用户选择的保存目标写入本地记忆，下次长按新增时默认选中上次目标。
- 点位保存后应立即渲染、聚焦并打开弹窗。

### KML 文件重命名

- 每个 KML 文件卡片提供重命名图标。
- 重命名使用项目统一弹窗组件。
- 空名称不保存。

### 右下角按钮收纳

- 默认仅保留定位、搜索、更多/图层等核心入口。
- 点击更多/图层按钮时显示图层控制，并显示 KML 管理和管理后台按钮。
- 再次点击更多/图层按钮时隐藏这些低频入口和图层控制。

### 访问密码

- 前端和后端访问密码最小长度统一为 4 位。
- 管理密码规则不变，仍至少 4 位。

## 验收标准

- 分享包含 `layer` 的链接后，接收方打开页面恢复同一图层。
- 新用户不导入 KML 文件，也能长按地图新增点位并保存到默认文件。
- 用户创建多个 KML 文件后，长按新增点位可选择目标文件，并记住上次选择。
- KML 文件可以新建、重命名、导入、导出、显隐和删除；默认文件不允许删除。
- 管理后台访问控制允许设置 4 位访问密码。
- 右下角 KML 管理和管理后台入口默认收起，点击更多/图层后出现。
- `npm run check`、`npm test`、`npm run build` 均通过。
