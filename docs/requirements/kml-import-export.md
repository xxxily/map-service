# KML 导入导出

## 背景和目标

地图需要支持导入外部 KML 文件，在当前底图上查看、管理和导出点、线、面要素。KML 标准坐标通常为 WGS84，经纬度直接叠加到高德底图时会出现位置偏差，因此需要提供坐标纠偏能力。

## 范围内事项

- 支持导入 `.kml` 文件中的 `Placemark` 点、线、面要素。
- 导入后默认开启坐标纠偏，将 WGS84 坐标按高德底图使用的 GCJ-02 坐标显示。
- 用户可以在导入前关闭默认纠偏，也可以对已导入的单个 KML 文件切换纠偏显示。
- 用户可以按 KML 文件启用或禁用显示；禁用后该文件的所有图层从地图移除，避免多个 KML 叠加造成界面凌乱或渲染压力过高。
- 内部存储和导出保持 KML 标准经纬度，不将 GCJ-02 显示坐标直接写入导出的 KML。
- 在纠偏开启时，新增点和拖拽点位需要先从地图显示坐标反算回 WGS84 后再保存。
- 点要素在地图上默认显示名称标签，名称过长时需要截断和限宽，避免遮挡地图主体内容。

## 范围外事项

- 不支持 KMZ 解压导入。
- 不保留 KML 原始样式、文件夹层级和扩展字段。
- 不实现复杂几何编辑，仅保留现有点位拖拽和属性编辑能力。

## 数据模型

每个导入的 KML 文件在本地保存：

- `id`：本地文件 ID。
- `name`：文件名。
- `coordCorrection`：坐标显示策略，`wgs84-to-gcj02` 表示开启纠偏，`none` 表示不纠偏。
- `enabled`：是否在地图上显示该 KML 文件，缺省按启用处理以兼容旧数据。
- `features`：要素数组，坐标均按 KML/WGS84 保存。

## 验收标准

- 使用高德底图时，默认导入的 WGS84 KML 点位应与底图位置对齐。
- 关闭纠偏后，同一 KML 文件按原始坐标显示。
- 对开启纠偏的文件新增或拖拽点位后，导出的 KML 仍能在 Google Earth 等标准 KML 工具中按正确位置使用。
- 导出文件不受当前纠偏开关影响，始终输出标准 KML 经纬度。
- 导入 KML 后无需点击点位图标，也能直接看到点位名称；长名称不会大面积覆盖地图。
- 禁用某个 KML 文件后，该文件的点、线、面和点位名称不再显示；再次启用后可以恢复显示，导出内容不受启用状态影响。
