# 对外开放图层 API 接口需求

## 1. 背景

在许多封闭的网络环境中，最终用户因为网络阻断或 DNS 污染，无法直接访问外部的公开或商业地图服务（如谷歌地图瓦片服务 `https://www.google.com/maps/vt?lyrs=s@189&gl=cn&x={x}&y={y}&z={z}`）。
本系统作为一个自建或部署于特定出口的地图中间件，目前已经支持前台地图渲染和离线缓存。为了将本系统的网络穿透与图层代理能力赋能给其他第三方系统或独立的 GIS 客户端，我们希望开放一个统一的、受控的对外图层代理 API 接口，并提供审计日志和安全控制。

---

## 2. 目标

- **对外提供标准化瓦片服务**：允许外部 GIS 客户端（如 Leaflet, OpenLayers, QGIS 等）通过本系统作为中转，拉取上游图层。
- **支持灵活配置的上游源**：管理员可自由定义上游瓦片 URL 模板，支持代理切换与 HTTP 代理中转。
- **安全准入控制**：支持基于 Token 的安全验证机制，支持随时启用/禁用，避免接口被恶意滥用。
- **轻量级请求审计**：提供请求日志记录，支持按上限（200 / 500 / 1000条）循环覆盖存储，保护系统性能，并防止隐私数据（如 Token 完整字符）泄漏。
- **管理后台可视化控制**：提供开关设置、安全配置、上游配置以及审计日志面板。

---

## 3. 范围

### 3.1 纳入范围
- **图层代理路由**：提供类似 `GET /api/v1/external/tile` 接口，支持通过模板参数解析与代理请求。
- **上游地址模板解析**：支持 `{x}`, `{y}`, `{z}` 占位符解析，替换为具体坐标后请求上游。
- **系统代理中转**：支持配置该图层接口是否走系统代理（复用已配置的代理服务器）。
- **安全 Token 校验**：支持配置项启用，强制在 URL 传入 `token=xxx`，且转发给上游前必须剥离该 Token 参数以防泄漏。
- **滚动日志审计**：
  - 记录包括：请求时间、访问 IP、User-Agent、请求坐标、响应状态码、耗时等。
  - 日志自动覆盖机制（先进先出），配置存储上限（200/500/1000 条）。
  - 日志脱敏（对记录的完整请求 URL 中的 Token 进行掩码处理）。
- **管理端 UI 面板**：增加图层 API 配置表单与访问日志列表视图。

### 3.2 暂不纳入
- **图层瓦片缓存（Cache）控制**：当前需求优先实现高透明度的实时反代，暂不对此对外接口进行深度缓存管理（未来可考虑引入与本地瓦片预缓存模块关联）。
- **分用户/多 Token 控制**：目前采用单 Token 准入模式，暂不支持多用户多 Token 细粒度管理。
- **高并发分布式限流**：图层接口限流暂不做分布式部署的扩展，仅做应用内防刷保护。

## 4. 功能需求

### F1. 图层代理路由与解析机制
- **路由端点**：`GET /api/v1/external/tile`
- **输入参数**：
  - `x`: 瓦片 X 坐标 (必填)
  - `y`: 瓦片 Y 坐标 (必填)
  - `z`: 瓦片 Z 坐标 (层级，必填)
  - `token`: 校验令牌 (若启用安全校验则必填)
- **转发逻辑**：
  - 替换配置的上游 URL 模板中的 `{x}`, `{y}`, `{z}`。
  - 剥离传入的 `token` 参数，保持其他可能需要的查询参数。
  - 根据开关决定是否使用配置的代理服务进行请求。
  - 若上游响应成功，将图像流以及正确的 `Content-Type` 头部（如 `image/png`, `image/jpeg`）透传回客户端。

### F2. 接口开放性与安全准入设置
- 管理员可在后台修改以下设置，并立即生效：
  - **启用状态** (`tileApi.enabled`)：布尔值。若为 `false`，则访问代理接口直接返回 `403 Forbidden`。
  - **上游图层接口地址** (`tileApi.upstreamUrl`)：字符串。默认为 `https://www.google.com/maps/vt?lyrs=s@189&gl=cn&x={x}&y={y}&z={z}`。
  - **是否使用代理** (`tileApi.useProxy`)：布尔值。指示是否通过本系统的全局代理配置拉取上游。
  - **Token 校验开关** (`tileApi.tokenEnabled`)：布尔值。
  - **准入 Token** (`tileApi.token`)：字符串。用于校验客户端身份。支持管理员重置生成随机的 Token。

### F3. 接口访问审计日志
- 每次外部接口被请求时，系统需异步在内存/轻量数据库中记录一条日志：
  - **请求时间**：ISO 格式时间戳。
  - **来源 IP**：发起请求的客户端 IP（需支持识别反向代理如 Nginx 传递的 `X-Forwarded-For` 或 `X-Real-IP`）。
  - **坐标参数**：格式如 `Z:{z} X:{x} Y:{y}`。
  - **脱敏请求 URL**：记录客户端的完整请求路径，但 `token=xxx` 部分必须转换为掩码形式（如 `token=abc****xyz`）。
  - **上游请求 URL**：实际向源站请求的 URL。
  - **User-Agent**: 客户端 UA。
  - **状态码**：上游响应状态码或本系统处理状态码（如 200, 401, 502）。
  - **响应耗时**：毫秒（ms）。
- **日志滚动清理**：
  - 存储上限配置项 (`tileApi.maxLogCount`)：支持 `200`、`500`、`1000` 三档选择，默认 `500`。
  - 当日志总数超过该上限时，系统自动剔除最老的一条记录（FIFO 队列）。
  - 提供“清空日志”的 API 和后台操作按钮。

### F4. 管理后台交互与日志看板
- **配置面板**：
  - 位于管理后台「设置」或独立新增的「对外 API」菜单中。
  - 包含图层接口总开关、上游配置、代理开关、Token 设置（包含生成/复制 Token 按钮）。
- **日志看板**：
  - 显示最近请求的日志列表（表格展示，支持按 IP、状态码过滤）。
  - 支持查看单条日志的详细 JSON 数据（包含 User-Agent 完整内容、耗时、上游详细返回错误）。
  - 提供一键清空日志按钮。

## 5. 非功能需求

### 5.1 安全性
- **Token 隐藏**：无论上游接口是否需要身份校验，传入本系统的 `token` 必须剥离，绝对不能透传给上游，防止内部敏感凭证泄漏给外部节点。
- **Token 脱敏日志**：日志持久化与后台显示时，Token 必须经过不可逆的字符掩码处理，且不能输出到系统 stdout 调试日志中。

### 5.2 性能与鲁棒性
- **无阻塞日志记录**：日志写入操作应异步进行，绝不阻塞瓦片图片流的传输。
- **内存保护**：日志上限硬性限制为最多 1000 条，防止由于日志无限增长导致 Node.js 内存溢出（OOM）。
- **超时保护**：反代上游时，设置合理的连接和读取超时时间（如 8 秒），超时后返回 504 错误，并释放连接，防止占用过多系统套接字。

## 6. API 说明

### 6.1 图层反向代理服务
- **路径**：`GET /api/v1/external/tile`
- **请求参数**：
  | 参数 | 类型 | 必须 | 说明 |
  | --- | --- | --- | --- |
  | `x` | 整数 | 是 | 瓦片 X 坐标 |
  | `y` | 整数 | 是 | 瓦片 Y 坐标 |
  | `z` | 整数 | 是 | 瓦片 Z 坐标（层级） |
  | `token` | 字符串 | 视配置 | 准入校验 Token（当 `tileApi.tokenEnabled` 为 `true` 时必填） |
- **响应**：
  - 成功：`200 OK`，`Content-Type: image/*`，返回瓦片图片二进制流。
  - 验证失败：`401 Unauthorized`，`Content-Type: application/json`，返回错误描述。
  - 接口禁用：`403 Forbidden`。
  - 上游不可达/出错：`502 Bad Gateway` 或 `504 Gateway Timeout`。

### 6.2 管理端日志管理
- **获取日志**：`GET /api/v1/admin/tile-api/logs` (需要管理员 Token)
- **清空日志**：`DELETE /api/v1/admin/tile-api/logs` (需要管理员 Token)

## 7. 验收标准

1. **基本代理验证**：通过配置谷歌地图作为上游，在 QGIS 或前端 Leaflet 中配置本接口 `/api/v1/external/tile?x={x}&y={y}&z={z}`，图层能够正常、流畅地渲染出图。
2. **代理切换校验**：启用代理时，请求能够成功通过系统配置的代理中转；关闭代理时，进行直连请求。
3. **安全 Token 验证**：
   - 启用 Token 时，不带 `token` 参数或传入错误 `token` 请求接口应返回 401，且不发起上游请求。
   - 携带正确 `token` 请求能成功获取图像，检查上游网络流量确认请求中已剥离 `token`。
   - 禁用 Token 时，不带 `token` 也能正常访问接口。
4. **日志滚动与覆盖**：
   - 产生的访问记录能实时显示在后台日志面板中。
   - 日志中的请求 URL 已对 Token 参数进行脱敏。
   - 设定上限为 200 条，发起 250 次请求后，日志总数保持在 200 条，且最早的 50 条记录已被自动清除。
   - 点击“清空日志”按钮，所有记录清空，面板恢复初始空状态。
5. **服务关闭机制**：关闭接口后，再次请求接口立即返回 403，且不生成除 403 记录外的任何多余上游网络请求。
