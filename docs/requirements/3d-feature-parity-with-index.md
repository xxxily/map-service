# 3D 地图与首页功能对标需求

## 背景

当前首页 `index.html` 已具备图层切换、定位、搜索、KML 数据管理、辅助线模式、管理后台入口与访问控制。`3d.html` 仅覆盖三维地球基础浏览、图层切换、自转、视角复位和返回 2D，用户在 3D 页面无法完成与首页一致的地图工作流。

## 目标

- `3d.html` 直接访问时必须遵循与首页一致的访问控制策略；后台开启访问控制时，用户需输入访问码后才能初始化三维地图。
- 3D 页面右下角工具入口与首页对齐，包含图层、KML、辅助线、搜索、定位、管理后台、返回 2D，以及 3D 独有的自转和视角复位。
- 3D 页面图层列表与首页图层名称、URL 参数和本地记忆保持一致，包含高德/Google 街道、卫星与高清对比图层。
- 3D 页面支持浏览器定位，并在三维地球上飞行到当前位置、显示定位标记。
- 3D 页面支持高德地点搜索，选中结果后飞行到对应位置并显示搜索标记。
- 3D 页面支持 KML 导入、导出、新建、重命名、显示隐藏、公共 KML 展示、公共编辑入口和点位新增/编辑/删除等核心工作流。
- 3D 页面支持辅助线模式，包括进入/退出、点击添加、选中删除、清空、撤销、重做和本地持久化。

## 约束

- 不直接编辑 `service/app/` 下构建产物。
- 前端提示、确认和输入必须复用 `src/ui/dialog.js`，不能使用浏览器原生阻塞弹窗。
- 3D 适配应复用现有 KML 数据格式、localStorage key 和 URL 状态，避免与首页形成两套互不兼容的数据。

## 验收

- 未开启访问控制时，直接打开 `/3d` 能正常进入三维地图。
- 开启访问控制时，直接打开 `/3d` 显示访问码输入页；输入正确访问码后才初始化三维地图。
- 从 3D 菜单可以打开管理后台 `/admin/overview`。
- 3D 图层菜单包含首页所有底图项，切换后 URL `layer` 参数和 `last_map_layer` 与首页兼容。
- 3D 搜索、定位、KML 面板和辅助线工具栏均能通过右下角菜单进入。
- 首页创建或导入的 KML 与辅助线刷新到 3D 后可见；3D 中新增的数据回到首页后也可见。
- 提交前运行 `npm run check`、`npm test`、`npm run build`。
