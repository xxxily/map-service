# 3D 地图交互与地形效果改进需求

## 背景

`3d.html` 已同步首页的搜索、定位、KML、辅助线、图层和访问控制入口，但在实际使用中暴露出 3D 场景下的交互和地形表现问题：

1. 辅助线模式点击后只出现中心点，没有可见十字线；再次点击中心点也没有删除按钮。
2. 移动端手势缩放变化率过低，放大/缩小需要很大的手势幅度。
3. 移动端没有键盘 `Shift`，无法像 PC 端一样进入自由调整倾角和旋转的 3D 操作状态。
4. 进入 `Shift` 3D 操作后，山势高度没有明显立体呈现，达不到类似 Google Earth 的 3D 观感。
5. Google 地图瓦片在 3D 中固定使用 1 倍图，没有参考 `src/map/layers.js` 按设备像素比选择 `scale`，高分屏下清晰度不如 2D 首页。

本文档只定义需求和解决方案，待审查通过后再进入实现。

## 问题分析

### 1. 3D 辅助线只显示点

当前 3D 辅助线使用 Cesium entity 绘制中心点和两条 `polyline`，线段按全球级长距离计算并设置 `clampToGround: true`。中心点可见，说明数据写入和 entity 创建基本可用；十字线不可见可能由以下因素共同导致：

- 线段跨越距离过长，部分线段落在地球背面或被地球/地形深度遮挡，低空视角下不符合用户对“当前视口辅助线”的预期。
- `clampToGround` 的 ground polyline 在材质、地形加载状态和浏览器 WebGL 能力上兼容性更敏感，虚线材质可能不能稳定呈现。
- 2D 辅助线按当前视口动态计算端点；3D 版目前按固定大地线长度计算，缺少随相机视口刷新的局部线段算法。
- 3D 版没有实现辅助线删除气泡；点击中心点只切换选中态，缺少与 2D 一致的“删除此辅助线”入口。

### 2. 移动端缩放变化率过低

当前代码主要优化了桌面端和 macOS 触摸板的 `wheel` 事件；移动端双指缩放大多由 Cesium 默认 `ScreenSpaceCameraController` 接管。默认控制器为了稳定性通常较保守，且现有最小高度、防穿透和相机约束会进一步削弱低空缩放体感。

需要为移动端建立独立、可调的 pinch 缩放曲线，而不是只依赖默认手势。

### 3. 移动端缺少 Shift 等价操作

PC 端 `Shift + 左键拖拽` 会进入手动 3D 操作逻辑，用于绕点击地面点旋转和调整俯仰角。移动端没有键盘修饰键，因此需要提供显式的模式切换控件。

建议在右上角增加一个 2D/3D 模式切换图标：

- 2D 模式：默认地图浏览模式，尽量保持上帝视角、稳定平移和缩放。
- 3D 模式：等价于“虚拟按住 Shift”，允许通过触摸手势旋转、调整倾角、缩放和移动。

### 4. 山势 3D 效果不明显

当前代码调用了 `Terrain.fromWorldTerrain()` 并设置 `terrainExaggeration = 2.2`，但实际立体感不明显，可能原因如下：

- Cesium World Terrain 通常依赖 Cesium Ion 授权；项目当前没有显式配置 Ion token，且代码重写了 `showErrorPanel`，用户侧不容易发现地形加载失败。若地形 provider 未成功加载，`terrainExaggeration` 不会产生可见效果。
- 地形是异步加载的，当前没有地形加载状态、失败提示或 provider 自检，无法判断是否真的从椭球体切换到了地形网格。
- 初始和常规视角接近垂直俯视；地形起伏只有在足够低空、倾斜视角和较高地形精度下才明显。
- 当前相机 pitch 被限制在 `-85°` 到 `-15°`，并且默认初始化为 `-90°` 俯视；用户如果没有进入低空斜视状态，山体会接近平面影像。
- 当前底图是 2D 瓦片影像。Google Earth 的“3D 感”不仅来自 DEM 地形，还来自 3D Tiles、摄影测量模型、建筑物模型、阴影、光照和高分辨率地形/影像组合。
- 没有启用面向 3D 浏览的光照、阴影、地形状态提示、典型山地区域快速入口，也没有把“是否加载真实地形”作为可验证状态暴露出来。

### 5. 3D Google 瓦片 scale 与 2D 不一致

`src/map/layers.js` 中 2D 图层会根据 `L.Browser.retina` 动态选择：

- Google 普通卫星和街道：`scale=2` 或 `scale=1`。
- Google HD 卫星：固定 `scale=3`。
- 高德卫星：按 retina 选择 `scl=2` 或 `scl=1`。

但 `src/3d.js` 当前内置的 Google 普通卫星、Google 街道、Google + 高德混合图层没有带 `scale` 参数，等价于长期请求 1 倍图；只有 HD 图层显式使用 `scale=3`。这会导致高 DPR 屏幕、移动端和 Retina Mac 上 3D 瓦片发虚，也会造成 2D/3D 图层体验不一致。

根因是 3D 页面维护了独立的 `layerSources`，没有复用或镜像 `layers.js` 的 scale 选择策略。

## 需求目标

### G1. 3D 辅助线可见且可管理

- 点击地图添加辅助线后，必须显示中心点和两条正交十字线。
- 十字线应按当前 3D 视口或局部切平面计算，表现为用户当前视野内的辅助线，而不是不可控的全球超长线。
- 点击辅助线中心点或线段后，应进入选中态并出现删除入口。
- 删除入口应复用项目统一弹窗/浮层体系，不使用 `alert`、`confirm`、`prompt`。
- 保持与 2D 辅助线共享 `map_guidelines` 数据结构，2D/3D 创建的数据互通。

### G2. 移动端缩放手感可用

- 移动端双指 pinch 缩放应明显响应，缩放变化率应可配置。
- 缩放应围绕手势中心或当前视口中心进行，避免用户放大后位置漂移过大。
- 保留地底穿透保护和最低相机高度限制。
- 缩放曲线需区分高空、中空和低空，避免高空过慢、低空过猛。

### G3. 移动端提供 2D/3D 显式模式切换

- 右上角增加 2D/3D 图标按钮，默认进入 2D 模式。
- 点击 3D 图标后进入 3D 模式，视觉上明确显示当前处于 3D 操作状态。
- 3D 模式下触摸手势建议映射：
  - 单指拖动：绕触点或屏幕中心旋转/调整倾角。
  - 双指 pinch：缩放。
  - 双指同向平移：平移地球视野。
  - 双指旋转：可选，优先用于 heading 旋转；如与平移冲突，则后续版本再启用。
- PC 端保留 `Shift` 入口，同时允许点击 2D/3D 按钮进入同等状态。
- 模式切换不能影响 KML、辅助线、搜索、定位等工具的点击可用性。

### G4. 真实地形与 3D 观感可验证

- 地形 provider 是否成功加载必须可验证，至少在控制台和 UI 状态中能看出“真实地形/平面椭球体 fallback”。
- 地形源应配置化，支持至少一种明确可用的生产方案：
  - Cesium Ion World Terrain：需要配置 token。
  - 自托管 Cesium terrain tiles：适合内网或离线部署。
  - 其他兼容 Cesium terrain provider 的地形服务：作为可选扩展。
- 当地形加载失败时，不应静默失败；应展示非阻塞状态提示，并回退到平面椭球体。
- 需要提供 3D 浏览推荐视角：低空、斜视、开启地形夸张，帮助用户快速验证山势。
- 需要评估是否引入 3D Tiles/建筑/摄影测量模型。仅 DEM 地形无法完全达到 Google Earth 城市级 3D 观感。

### G5. 3D 图层 scale 与首页一致

- 3D Google 普通瓦片必须按设备像素比选择 `scale=2` 或 `scale=1`，与 `src/map/layers.js` 保持一致。
- 3D Google HD 图层继续固定使用 `scale=3`。
- 3D 高德卫星图层应评估是否也按 2D 逻辑使用 `scl=2` 或 `scl=1`，避免高分屏下 2D/3D 清晰度不一致。
- 图层名称、URL 参数和 localStorage 中的 `last_map_layer` 仍保持与首页兼容。
- 建议抽出共享图层 URL 构造逻辑，减少 2D/3D 各维护一份配置造成的漂移。

## 推荐方案

### 方案 A：修正 3D 辅助线绘制和删除交互

1. 将 3D 辅助线从“全球固定长线”改为“相机视口局部线”：
   - 以点击点为中心建立局部 ENU 切平面。
   - 根据当前相机高度和视口范围计算线段长度。
   - 线段随相机移动和缩放刷新端点。
2. 渲染策略：
   - 优先使用普通 `polyline` + 小高度偏移，减少 ground polyline 兼容性风险。
   - 必要时关闭深度测试或设置足够的 `disableDepthTestDistance`，确保辅助线作为工具层可见。
   - 选中态使用颜色和线宽区分。
3. 增加 3D 辅助线删除浮层：
   - 点击中心点或线段后，在屏幕坐标位置显示小浮层。
   - 浮层包含删除按钮。
   - 支持 `Delete` / `Backspace` 删除选中辅助线。

### 方案 B：移动端缩放增强

1. 新增移动端 touch/pointer gesture 管理器。
2. 记录双指距离变化，使用非线性缩放曲线：
   - 高空：每次 pinch 按高度的较大比例移动。
   - 中空：线性平滑过渡。
   - 低空：降低速度并加强地表碰撞保护。
3. 将缩放灵敏度、最低高度和最大单次位移抽成常量，便于调试。
4. 保留 Cesium 默认控制器作为 fallback，避免不支持 Pointer Events 的浏览器无法操作。

### 方案 C：右上角 2D/3D 模式切换

1. 在右上角图层控件附近增加模式按钮。
2. 状态机：
   - `mode = "2d"`：常规浏览，限制 pitch，交互偏稳定。
   - `mode = "3d"`：启用虚拟 Shift，触摸和鼠标拖动进入自由 3D 操作。
3. 模式切换后同步 CSS 状态，提供清晰视觉反馈。
4. 移动端 3D 模式下优先实现单指旋转/倾角 + 双指缩放；双指平移和双指旋转在手势冲突验证后再启用。

### 方案 D：地形与 Google Earth 类观感增强

1. 增加地形配置：
   - `terrain.enabled`
   - `terrain.provider`
   - `terrain.ionToken`
   - `terrain.url`
   - `terrain.exaggeration`
2. 增加地形加载状态：
   - 正在加载真实地形。
   - 已启用真实地形。
   - 地形加载失败，已回退平面模式。
3. 在 3D 模式下提供推荐相机参数：
   - 山区低空高度范围。
   - 默认 pitch 从纯俯视调整到可见山势的斜视角。
   - 可选地形夸张滑块或预设档位。
4. 增加真实地形自检：
   - 加载后在典型山地区域采样高度。
   - 如果采样高度长期为 0 或 provider 加载失败，标记为地形不可用。
5. 后续增强项：
   - 可选 3D Tiles 建筑/摄影测量图层。
   - 光照、阴影、雾效、大气和天空盒优化。
   - 山区演示点或“飞到山地测试区”调试入口。

### 方案 E：统一 2D/3D 瓦片 scale 策略

1. 抽出共享的设备像素比判断：
   - 不能在 3D 中直接依赖 Leaflet 的 `L.Browser.retina`。
   - 可使用 `window.devicePixelRatio > 1` 或封装 `getTileScale()`。
2. 抽出共享的瓦片 URL 构造：
   - Google 普通卫星：`scale=${googleScale}`。
   - Google 街道：`scale=${googleScale}`。
   - Google HD 卫星：`scale=3`。
   - 高德卫星：`scl=${autonaviScale}`。
   - 高德街道维持当前 `scl=1`，除非验证高分屏街道文字也需要 `scl=2`。
3. 让 `src/map/layers.js` 和 `src/3d.js` 共享同一套图层 URL 常量或工厂函数。
4. 保持 3D 的 Cesium `UrlTemplateImageryProvider` 适配层只负责 provider 创建，不再硬编码 scale 差异。

## 验收标准

### 辅助线

- 在 3D 页面进入辅助线模式后，点击地图能看到中心点和完整十字线。
- 缩放、旋转、调整倾角后，辅助线仍在当前视口中可见。
- 点击辅助线中心点或线段出现删除入口。
- 删除、清空、撤销、重做可用。
- 2D/3D 辅助线数据互通。

### 移动端缩放

- 移动端双指缩放在 2 到 3 次手势内可以从城市级缩放到街区级，反向也能顺畅缩小。
- 低空缩放不穿透地形或地球。
- 缩放中心与用户手势中心保持合理一致。

### 2D/3D 模式切换

- 移动端可以通过右上角图标进入 3D 操作状态。
- 3D 状态下可通过手势调整 heading、pitch、zoom。
- 再次点击可回到 2D 稳定浏览状态。
- PC 端 `Shift` 逻辑不回退。

### 地形效果

- 地形加载成功/失败状态可见。
- 配置有效地形源后，在山地区域低空斜视能明显看到山体起伏。
- 地形加载失败时，页面仍可使用，并明确提示当前为平面 fallback。
- 文档说明仅 DEM 地形与 Google Earth 完整 3D 城市场景的差异，避免误判预期。

### 图层清晰度

- 在 Retina Mac、常见高 DPR 手机和普通 1x 屏上，3D Google 普通卫星/街道请求的 `scale` 与首页一致。
- `谷歌高德/卫星（HD）` 在 3D 中继续请求 `scale=3`。
- 3D 与 2D 切换同一图层时，主观清晰度不应出现明显倒退。
- URL `layer` 参数和 `last_map_layer` 不因 scale 适配发生兼容性变化。

## 非目标

- 本阶段不直接实现完整 Google Earth 同等级全球摄影测量模型。
- 本阶段不强制引入新的付费地形或 3D Tiles 服务。
- 本阶段不重构所有 3D 相机控制逻辑，只围绕移动端可用性和地形可见性做增强。

## 待审查问题

- 是否允许配置 Cesium Ion token，还是必须优先支持自托管地形？
- 右上角 2D/3D 模式按钮是否与现有图层按钮并列，还是并入更多工具菜单？
- 移动端 3D 模式下，单指拖动优先“旋转/倾角”还是“平移”？
- 辅助线在 3D 中应作为始终可见的工具层，还是应严格贴合地形并允许被地形遮挡？
- 是否需要增加 3D Tiles/建筑图层作为后续阶段目标？
- 高德街道图层是否也需要在 3D 中跟随 DPR 使用 `scl=2`，还是继续保持 2D 当前的 `scl=1` 策略？

## 建议实施顺序

1. 修复 3D 辅助线可见性和删除入口。
2. 统一 2D/3D 图层 scale 策略，修复 Google 瓦片 1 倍图问题。
3. 增加移动端 pinch 缩放增强。
4. 增加右上角 2D/3D 模式切换。
5. 增加地形 provider 配置、加载状态和失败提示。
6. 调整 3D 推荐视角、地形夸张和光照参数。
7. 评估 3D Tiles/建筑/摄影测量数据源。
